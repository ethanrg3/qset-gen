The three loops

Plan §3. Each loop is independent: session ingestion, set generation, and set submission all converge on the same Student.weak_skills/strong_skills state. Green nodes are implemented; amber nodes need Notion DB IDs or env vars.

Session loop

Fires after each tutoring session via qset-gen ingest-session. Writes one row to Session Signals; recomputes weak/strong.

Fathom transcript → extractor.py (Anthropic, injectable client) → Session Signals DB → adapt/weak_strong.py → Student.weak/strong

Generation loop

Fires per homework request via qset-gen generate. Runs entirely off the local SQLite cache — no Notion roundtrip per set.

Notion DBs → cache.py (SQLite) → scoring.py (6 signals) → constraints.py (floors + no-streak) → render/ (self-contained HTML)

Submission loop

Student opens the HTML, completes it, clicks Submit. The HTML POSTs to the FastAPI webhook with a Bearer token baked in at render time.

HTML SPA → webhook/app.py /submit → Q-History DB → adapt/weak_strong.py → Student.weak/strong

Notion data model

Six databases. Three exist; three need creating; the existing Students DB likely needs some new properties.

Questions exists

Already in your workspace. Fields used: question_id (Title), test, section, skill_tag (Relation), difficulty_low/difficulty_high, html_render, answer_key, explanation_html, time_target_sec, active.

Students augment

Exists; confirm these properties: current_act_math, target_act_math, test_date, weak_skills (Relation→Skill Taxonomy, multi), strong_skills (Relation, multi), last_set_generated_at, last_session_at.

Skill Taxonomy exists

The 75-skill ACT math taxonomy. Fields: skill_id (Title), name, description.

Q-History create

One row per attempt. Fields: attempt_id (Title), student (Relation), question (Relation), attempted_at (Date+time), correct (Checkbox), time_spent_sec (Number), set_id (Text), confidence (Select).

Session Signals create

One row per tutoring session. Fields: session_id (Title), student (Relation), session_date (Date), duration_min, skills_practiced (Relation, multi), skills_struggled (Relation, multi), skills_introduced (Relation, multi), skills_mastered_today (Relation, multi), misconceptions (Text), homework_directives (Text), raw_transcript_excerpt (Text), extraction_model (Text).

Skill Status History create

Append-only audit log of weak/strong transitions. Fields: change_id (Title), student (Relation), skill (Relation), prior_status (Select), new_status (Select), weakness_score (Number), triggered_by (Text), logged_at (Date).

Next steps — your side

In order. Each step is independently small and reversible. After step 5 the system is end-to-end live.

  1. Provision the three new Notion databases

    Create Q-History, Session Signals, and Skill Status History with the schemas above. Use the same workspace as Questions/Students/Skill Taxonomy.

    ~15 minutes. Lift the field lists from the Notion data model section above.

  2. Audit the Students DB

    Confirm the seven properties listed on the Students card exist with the right types. Add any that are missing.

    ~5 minutes.

  3. Create a Notion integration and share databases

    Go to notion.so/my-integrations → New integration (internal, no user info). Copy the token. Then on each of the six DBs: … menu → Connections → add your integration.

    ~5 minutes.

  4. Fill in .env at the project root

    cp .env.example .env
# then edit:
NOTION_TOKEN=ntn_xxxxxxxxxxxxx
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxx
NOTION_DB_QUESTIONS=<32-char id from URL>
NOTION_DB_STUDENTS=<...>
NOTION_DB_Q_HISTORY=<...>
NOTION_DB_SESSION_SIGNALS=<...>
NOTION_DB_SKILL_TAXONOMY=<...>
NOTION_DB_SKILL_STATUS_HISTORY=<...>
WEBHOOK_BASE_URL=https://<your-host>
WEBHOOK_SECRET=<long random string>

    The DB ID is the 32-char string in the database URL between the workspace slug and ?v=.

    ~5 minutes once you've copied the IDs from URLs.

  5. Pick a webhook host

    Easiest options, ranked by setup time:

    Cloudflare tunnel (~5 min, free, no account): brew install cloudflared, then cloudflared tunnel --url http://localhost:8787. Use the printed URL as WEBHOOK_BASE_URL.

    Fly.io (~15 min, free tier): fly launch from the repo.

    Render (~15 min): push to GitHub, point Render at qset_gen.webhook.app:app.

    After this step the rendered HTML can actually POST results.

  6. Hand off three things for the NotionGatewayLive wiring

    When you come back to plug in the live gateway, I need from you:

    1. A sample Question's raw Notion page (so I can read skill_tag correctly — is it a single Relation, a multi-Relation, etc.).

    2. Confirmation that the Title of each Question is the human-readable ID like ACTM-0421 (the models assume this; happy to flip to UUID-based).

    3. Optional: one sanitized Fathom transcript so we can write the extraction snapshot test that locks the prompt against drift.

    After that, NotionGatewayLive becomes ~150 LOC and the system goes live.
